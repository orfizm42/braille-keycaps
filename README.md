# braille-mx-keycaps

MX互換メカニカルキーボード向け、点字付き3Dプリントキーキャップジェネレーター。

---

## 概要

Cherry MX互換スイッチに対応した点字付きキーキャップのSTEPデータです。メカニカルキーボード愛好家・Makerコミュニティ・研究・教育目的での利用を想定しています。

STEPデータはGitHubで無料公開しています。

---

## 特徴

- **62キーフルセット** — アルファベット・数字・ファンクション・修飾・ナビゲーションキーに対応
- **JIS T 0921 / UNESCO準拠の点字ドット** — 直径1.5mm・突出高さ0.5mm・セル間隔2.5mm
- **特殊キー識別リブ** — ファンクション・修飾キーなどの上面左端に縦リブ（高さ0.7mm・幅1.5mm）を配置。点字と混同しない形状で通常キーと触覚的に即座に区別できる設計。
- **OPKベース** — Matt3o氏のOPK（Open Programmatic Keycap、Apache 2.0）をベースに改変

---

## 点字パターンについての注意

特殊キー（Ctrl・Alt・Escなど）の点字パターンは、各キー名称の頭文字に対応するUEB（統一英語点字）Grade 1パターンを割り当てた**独自定義**です。国際標準規格による特殊キーの点字表記は存在しないため、実用目的での使用前に点字の専門家・当事者による監修を推奨します。

## 印刷について

点字ドットの突出高さが0.5mm・リブが0.7mmと微細なため、**レジン3Dプリンターを推奨**します。FDMプリンターでは精度が不足する可能性があります。推奨レジン・印刷精度については現在検証中のため、試作結果をもとに随時更新します。

## 制作方法について

本プロジェクトはCadQueryライブラリとOPKを組み合わせて実装しました。主な技術的課題と解決方法は以下のとおりです。

**点字ドットの曲面追従**  
キーキャップ上面にはスクープ（くぼみ）があるため、バウンディングボックスのzmax（リム最高点）を基準にするとドットが空中に浮いてしまいます。OCCTのレイキャストAPIを用いて各ドット位置の真上から実際の表面Zを取得し、全てのドットを正確に表面から突出させました。

**ドーム形状の数学的導出**  
底面直径1.5mm・突出高さ0.5mmのドームを球でunionする際の球半径Rを `R = (r² + h²) / (2h)` から導出しています（R ≈ 0.8125mm）。これにより球をunionするだけで自動的に正しいドーム形状が得られます。

**特殊キー識別リブ**  
既存の点字キーボード製品は特殊キーをキー自体の横幅を広げて複数セルの略語で表記していますが、本プロジェクトでは1Uサイズを維持したまま、上面左端に縦リブを配置する独自アプローチを採用しました。リブは点字ドット（球）と同原理で、レイキャストで得た表面Zの真上に直方体を配置しています。

**生成時間の最適化**  
`opk.keycap()`はOCCTのboolean演算を複数回実行するため1呼び出しあたり約2分かかります。同じ`(unitX, unitY, angle)`のキーはベースキャップをキャッシュして使い回すことで、フルセット62キーでの`opk.keycap()`呼び出しを50回から8回に削減しました。
